# Internal Design Notes

These numbered documents (`01-PRD.md` through `18-HOOKS-AND-GUARDRAILS.md`)
are the original design notes that drove the early phases of this project.
They were authored when the repository was a private workspace and assumed
an internal audience.

They are kept here for historical context and as a reference for
maintainers, but they are not the customer-facing documentation. **The
canonical user docs live under [`docs/user/`](../../user/).**

If you are a customer or a first-time reader, start with:

- [README](../../../README.md) at the repository root
- [User documentation index](../../user/README.md)
- [Quick start](../../user/quickstart.md)
- [Customer alpha checklist](../../user/customer-alpha-checklist.md)

If you are a maintainer onboarding to the project, read these in number
order; they describe how the system was originally specified, and most of
the architecture decisions (now formalised in
[`docs/architecture/decisions/`](../../architecture/decisions/)) trace
back to them.
